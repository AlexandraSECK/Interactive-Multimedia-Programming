Questions :

1. RWD

Question 1: What is the purpose of the RWD? Why it is used? How things are done before RWD? What is the difference?

Answer 1: The purpose is to have a Website with an adaptable size of screen and adaptable in every device. Before there were website for specified device. Ex: « iphone website ». The Difference in RWD each device not treated separately it the same website forevery device.

Question 2: How css media queries can help you building responsive web sites?

Answer 2: Media queries help to adapt the website according to the device and the size of the screen or other physical characteristics.

Question 3: Why to use em (font) based sizing for example margins?

Answer 3: Em is relative to the size of the font. Use em permits to have margins proportional to the size of the font and more adapted to the device (size).

Question 4: Article is already 7 years old, do we have any new technology which can be used to achieve same type of UI responsiveness as described in the article. Buzz words: (CSS Grid, Flexbox)

Answer 4: Since 2009 it's possible to use Flexbox. It is usefull for responsive design because heights and the widths are flexible, it's possible to aligned elements or choose the space beetween elements. It helps to adapted the site to the size of the screen and it uses less code. CSS grid permits to choose the order of the elements and to create a grid adapted to the space and so the size of the screen. The user can create a structure for their website and make it responsive easlily.

3. RWD in practice

Question: How could you prevent user zooming the site on mobile?

Answer: It's possible to use this :
<meta name="viewport" content="max_width=480,initial-scale=1, maximum-scale=1" >
It will prevent the user zooming because of the maximum scale that is equal to the initial scale. It seems to work only on mobile.
